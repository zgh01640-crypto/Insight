# Insight 智能分析助手 — 设计方案

## Context

用户希望在现有经营分析系统上叠加一个 AI 对话层，允许以自然语言提问并获得数据驱动的分析结果（只读）。交互形式为全局浮动对话框，不破坏现有页面结构。

---

## 架构方案：Tool Use + 后端代理

### 整体链路

```
用户自然语言提问
      ↓
前端浮动对话框（Insight 前端）
      ↓  POST /api/ai/chat  (流式 SSE)
后端 AI 路由（新增 routers/ai.py）
      ↓  Anthropic SDK 调用 Claude
Claude 决策调用哪些 Tool
      ↓  后端直接调用已有 dashboard/target/opp 函数
拿到真实数据后 Claude 生成分析回答
      ↓  SSE 流式返回前端
前端逐字渲染回答
```

### 为什么用 Tool Use 而不是把数据塞 prompt

- 数据量大（年度总览有 4×3=12 条指标），每次全量塞入浪费 token
- Claude 可按需取数据，多轮追问只调用必要接口
- 后端可直接复用已有函数，无需重写查询逻辑

## 后端实现

### 1. 新增依赖（requirements.txt）

```
openai>=1.50.0    # DeepSeek 兼容 OpenAI 接口，直接用 openai SDK
```

### 2. 环境变量

在 `docker-compose.yml` 的 backend 服务中添加：
```yaml
environment:
  - LLM_API_KEY=${LLM_API_KEY}
  - LLM_BASE_URL=https://api.deepseek.com
  - LLM_MODEL=deepseek-chat
```

本地在项目根目录创建 `.env` 文件（已在 .gitignore）：
```
LLM_API_KEY=sk-...
LLM_BASE_URL=https://api.deepseek.com
LLM_MODEL=deepseek-chat
```

未来切换其他模型只改环境变量，代码不变。

### 3. 新文件：`backend/routers/ai.py`

**路由：** `POST /api/ai/chat`（SSE 流式响应）

**请求体：**
```json
{
  "messages": [
    {"role": "user", "content": "大数据事业部今年合同完成率是多少？"}
  ],
  "year": 2026
}
```

**Tool 清单（共 5 个，覆盖所有只读数据场景）：**

| Tool 名称 | 映射函数 | 说明 |
|-----------|---------|------|
| `get_overview` | `dashboard.overview()` | 所有事业部 YTD 达成率+同比 |
| `get_division` | `dashboard.division_detail()` | 单个事业部月度数据+缺口 |
| `get_quarterly` | `dashboard.quarterly_dashboard()` | 季度汇总（含各事业部） |
| `get_monthly` | `dashboard.monthly_dashboard()` | 月度汇总（含各事业部） |
| `get_opp_support` | `dashboard.opportunity_support()` | 商机分析（漏斗+完成率排名） |

**系统提示词（System Prompt）要点：**
- 角色：产品中心经营分析助理
- 约束：只查询提供的工具，不编造数据；金额单位万元；涉及比率保留1位小数
- 上下文注入：当前年份、当前月份、4个事业部名称列表

**实现方式：** `agentic_loop` 模式 —— 循环调用 Claude 直到无 tool_use，每次 text delta 通过 SSE 推送。

### 4. 在 `main.py` 注册路由

```python
from routers import ai
app.include_router(ai.router, prefix="/api/ai", tags=["AI助手"])
```

## 前端实现

### 1. 新增 API 方法（`src/api/index.js`）

```javascript
// 流式聊天（返回原始 Response，由组件处理 SSE）
export const chatWithAI = (messages, year) =>
  fetch('/api/ai/chat', {
    method: 'POST',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify({ messages, year })
  })
```

### 2. 新文件：`frontend/src/components/AIChat.vue`

**功能：**
- 右下角悬浮按钮（sparkle 图标），点击展开/收起
- 对话气泡列表（user 右对齐，assistant 左对齐）
- 输入框 + 发送按钮（Enter 发送）
- SSE 流式接收，逐字渲染 assistant 回答
- Markdown 基础渲染（粗体、换行）
- loading 状态（打字动画 `...`）

**状态：**
```javascript
const open = ref(false)
const messages = ref([])  // { role, content }
const input = ref('')
const thinking = ref(false)
```

### 3. 在 `App.vue` 挂载

```html
<AIChat />  <!-- 放在模板最底部，与路由视图平级 -->
```

## 验证方式

1. `docker-compose up --build` 重新构建后端
2. 打开 http://localhost:5173，右下角出现悬浮 AI 按钮
3. 点击展开，输入"产品中心今年合同完成率是多少"，应得到带数据的回答
4. 追问"哪个事业部表现最差"，Claude 应基于上一轮已有数据作答或调用工具补充
5. 查看 http://localhost:8010/docs 中 `/api/ai/chat` 端点文档

## 说明

- **API Key 管理**：开发时通过 `.env` + `docker-compose.yml` 注入，不写入代码
- **模型选择**：使用 DeepSeek（`deepseek-chat`），通过 OpenAI 兼容接口接入，环境变量可切换任意兼容模型
- **费用控制**：仅 tool 调用时才拉取数据，聊天历史默认保留最近 10 轮
- **无数据库存储**：对话历史仅存于前端内存，刷新即清空（简化架构）
