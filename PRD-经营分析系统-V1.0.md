## 1. 产品概述

### 1.1 背景

产品中心下辖五个事业部：**智能建造事业部**、**大数据事业部**、**数字交易事业部**、**智慧政务事业部**、**创新业务事业部**。每个事业部年初设定合同、收入、回款三项年度指标目标，并将目标分解至每月。

当前经营数据分散在多个 Excel 文件中，缺乏统一的汇总分析平台，导致：
- 各事业部完成进度对比依赖人工汇总，效率低
- 商机与目标的支撑关系无法直观呈现
- 历史数据追溯困难，趋势分析缺失

### 1.2 产品目标

构建一个面向产品中心负责人的 **经营分析系统（Insight）**，实现：

1. **目标管理**：集中维护各事业部年度目标及月度分解
2. **数据沉淀**：通过 Excel/CSV 导入积累月度完成情况数据
3. **商机跟踪**：记录各事业部季度商机明细，支撑目标预测
4. **经营分析**：可视化呈现目标达成率、月度趋势、商机漏斗等核心分析视图

### 1.3 核心价值

| 价值点 | 说明 |
|--------|------|
| 一站式数据中心 | 所有事业部的目标、完成、商机数据统一管理 |
| 实时达成率追踪 | 随时查看各事业部、各指标的 YTD 进度 |
| 商机支撑可视化 | 商机金额汇总与目标缺口对比，预判全年达成可能性 |
| 历史趋势分析 | 多年度、跨月度的经营走势，辅助管理决策 |

### 1.4 版本范围

本 PRD 覆盖 **V1.0** 版本，定位为 MVP（最小可行产品），聚焦核心数据录入与分析功能。

## 2. 用户角色与权限

### 2.1 角色定义

本系统 V1.0 仅服务于**单一用户角色**：

| 角色 | 描述 | 权限范围 |
|------|------|----------|
| 产品中心负责人 | 系统唯一使用者，对全部数据拥有完整操作权限 | 目标设置、数据导入、商机录入/编辑/删除、看板查看、模板下载 |

> **V1.0 不设多用户登录体系**。系统可配置简单的访问密码保护，防止误操作或未授权访问。

## 3. 功能模块详细需求

### 3.1 模块一：年度目标管理

#### 3.1.1 功能描述

管理各事业部每个年度的三项指标（合同、收入、回款）目标值，以及目标到月度的分解计划。

#### 3.1.2 功能清单

**年度目标设置**

- 支持按年份（如2025、2026）切换管理视图
- 展示5个事业部 × 3项指标的目标矩阵（15个目标值）
- 每个目标值支持在线编辑（点击单元格直接修改）
- 保存时记录修订时间（目标变更留痕）
- 支持通过 Excel 导入批量设置年度目标

**月度目标分解**

- 在年度目标详情页，展开某事业部后可查看/编辑12个月的目标分解值
- 约束：各月分解值之和 = 年度目标值（系统自动校验，超出或不足时给出提示，但不强制阻止保存）
- 支持通过 Excel 导入月度分解数据
- 提供"均摊"快捷操作：一键将年度目标平均分配至12个月

**目标修订记录**

- 每次修改年度目标时，系统记录：修改时间、原值、新值
- 提供变更历史查看入口

#### 3.1.3 交互逻辑

```
年度目标管理页面
├── 顶部：年份选择器（默认当前年）
├── 主体：目标矩阵表格
│   行：5个事业部
│   列：合同目标、收入目标、回款目标
│   每格：年度目标值（可点击编辑）
├── 操作区：[批量导入] [下载模板] [查看变更记录]
└── 展开事业部行 → 显示12个月分解值行
```

#### 3.1.4 字段说明

| 字段名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| 年份 | Integer | 是 | 如 2026 |
| 事业部 | Enum | 是 | 固定5个事业部 |
| 指标类型 | Enum | 是 | 合同/收入/回款 |
| 年度目标值 | Decimal | 是 | 单位：万元 |
| 月度分解值（1-12月） | Decimal×12 | 否 | 单位：万元 |

### 3.2 模块二：月度数据导入与管理

#### 3.2.1 功能描述

每月通过 Excel/CSV 导入各事业部实际完成数据，系统存储后用于看板分析与同比增长率计算。

> **多年度数据策略**：每次导入文件可包含任意年份的历史数据（如在导入2026年数据时，同时包含2025年及更早年份的数据）。系统按"事业部+指标+年月"覆盖写入，无论数据属于哪个年份均统一处理。这样系统在首次导入时即可建立历史数据基线，从而支撑同比增长率等跨年度分析。

#### 3.2.2 功能清单

**数据导入**

- 支持 `.xlsx`、`.xls`、`.csv` 格式
- 导入前展示字段映射预览（用户确认后再写入数据库）
- 导入规则：
  - 同一事业部+指标+年月组合，**覆盖写入**（而非追加）
  - 导入前提示"本次将覆盖 X 条已有数据，是否继续"
- 导入结果反馈：成功X条、失败X条、失败原因列表（可下载）

**数据查看与管理**

- 以月为单位的完成数据列表：事业部 × 指标 × 年月 × 完成值
- 支持按年份、月份、事业部过滤
- 支持单条数据在线编辑（修正录入错误）
- 支持单条数据删除

**导入历史**

- 记录每次导入操作：时间、文件名、导入数量、状态
- 支持查看历史导入详情

#### 3.2.3 导入文件格式要求

标准导入模板包含以下列：

| 列名 | 说明 | 示例 |
|------|------|------|
| 年份 | 4位数字 | 2026 |
| 月份 | 1-12的整数 | 3 |
| 事业部 | 与系统内名称完全一致 | 智能建造事业部 |
| 指标类型 | 合同/收入/回款 | 合同 |
| 完成值 | 数值，单位万元 | 1250.5 |

### 3.3 模块三：商机管理

#### 3.3.1 功能描述

记录各事业部按季度提交的商机明细，支撑目标达成预测与过程管理。

#### 3.3.2 功能清单

**商机录入与编辑**

- 支持 Excel/CSV 导入批量录入商机
- 支持在系统内手动新增、编辑、删除单条商机
- 支持商机状态更新（跟踪商机推进进度）

**商机查看**

- 默认列表视图，支持以下过滤：
  - 事业部（多选）
  - 所属年度
  - 所属季度（Q1/Q2/Q3/Q4）
  - 指标类型（合同/收入/回款）
  - 商机阶段
  - 商机状态
- 支持按金额、预计达成时间排序
- 汇总行：当前筛选条件下的商机总金额

#### 3.3.3 商机字段定义

| 字段名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| 商机名称 | String | 是 | 项目/客户名称 |
| 所属事业部 | Enum | 是 | 5个事业部之一 |
| 指标类型 | Enum | 是 | 合同/收入/回款（该商机归属的指标口径） |
| 所属年度 | Integer | 是 | 商机计划达成年度 |
| 所属季度 | Enum | 是 | Q1/Q2/Q3/Q4（该商机提交的季度） |
| 预计金额 | Decimal | 是 | 单位：万元 |
| 预计达成时间 | Date | 否 | 预计签约/确认收入/回款的时间 |
| 商机阶段 | Enum | 是 | 线索 / 立项 / 报价 / 签约跟进 / 已完成 |
| 商机状态 | Enum | 是 | 进行中 / 已赢单 / 已输单 / 已搁置 |
| 备注 | Text | 否 | 补充说明 |
| 创建时间 | DateTime | 自动 | 系统自动记录 |
| 最后更新时间 | DateTime | 自动 | 系统自动记录 |

#### 3.3.4 商机导入模板格式

| 列名 | 必填 | 示例 |
|------|------|------|
| 商机名称 | 是 | 某市智慧城市项目 |
| 所属事业部 | 是 | 智慧政务事业部 |
| 指标类型 | 是 | 合同 |
| 所属年度 | 是 | 2026 |
| 所属季度 | 是 | Q2 |
| 预计金额（万元） | 是 | 800 |

### 3.4 模块四：经营分析看板

#### 3.4.1 功能描述

以可视化图表呈现经营数据，支持多维度分析，为产品中心负责人的经营决策提供数据支撑。

#### 3.4.2 看板页面组成

##### 视图一：总览仪表盘

**定位**：首页/默认视图，快速掌握全局经营状况

**组件列表**：

| 组件 | 说明 |
|------|------|
| 年度/月份选择器 | 控制全页数据维度（当前年度 + 已到月份） |
| 全中心 KPI 汇总卡片 | 合同/收入/回款三项指标的YTD完成值、目标值、完成率，以及**与上年同期的同比增长率** |
| 事业部完成率对比图 | 5个事业部 × 3项指标的完成率横向柱状图，直观对比 |
| 事业部进度热力表 | 表格形式：行=事业部，列=指标，格内显示完成率，颜色标注（绿≥80% / 黄60-79% / 红<60%） |
| 同比增长率卡片 | 各事业部本年YTD完成值 vs 上年同期完成值，展示增长率（正增长绿色，负增长红色） |
| 本月新增完成 | 本月刚录入的完成值变化摘要 |

**完成率计算逻辑**：
- **月度完成率** = 本月实际完成值 / 本月目标分解值 × 100%
- **YTD完成率** = 年初至今累计完成值 / 年初至今累计目标分解值 × 100%
- **全年预测完成率** = 年初至今累计完成值 / 年度目标值 × 100%（进度参考）
- **同比增长率** = (本年YTD完成值 - 上年同期完成值) / 上年同期完成值 × 100%（需系统内有上年历史数据）

##### 视图二：单事业部深度分析

**定位**：从总览钻取，详细分析单个事业部的经营表现

**触发方式**：点击总览中的事业部名称

**组件列表**：

| 组件 | 说明 |
|------|------|
| 事业部名称 + 年度选择 | 页面顶部 |
| 三项指标 KPI 卡 | 合同/收入/回款的年度目标、YTD完成、完成率、**同比增长率** |
| 月度趋势折线图 | X轴=月份，Y轴=完成值，三条线分别代表合同/收入/回款；叠加目标分解线（虚线）；支持切换"本年 vs 上年"双年对比模式 |
| 月度完成率柱状图 | 每月完成率柱形图，低于阈值（**60%**）的月份标红 |
| 目标缺口分析 | 当前累计完成 vs 年度目标，剩余月份平均需完成金额 |
| 本事业部商机汇总 | 当前年度进行中商机总额，与目标缺口对比 |

##### 视图三：商机支撑分析

**定位**：分析在手商机能否覆盖目标缺口

**组件列表**：

| 组件 | 说明 |
|------|------|
| 年度 + 指标类型选择器 | 过滤商机 |
| 目标达成概览 | 年度目标值、YTD完成值、目标缺口 |
| 商机覆盖率 | 进行中商机总金额 / 目标缺口 × 100%（乐观/保守两种口径） |
| 商机阶段漏斗图 | 线索→立项→报价→签约的商机数量与金额漏斗 |
| 季度商机分布 | 按Q1/Q2/Q3/Q4的商机金额堆叠柱状图 |
| 商机明细表 | 支持在看板内直接查看筛选后的商机列表（快照） |

##### 视图四：同比/趋势分析

**定位**：跨年度比较，识别经营趋势与增长规律（导入多年历史数据后即可使用）

**组件列表**：

| 组件 | 说明 |
|------|------|
| 年度同比柱状图 | 选定的指标（合同/收入/回款）按年度比较累计完成值，标注同比增长率 |
| 月度走势多年对比 | 同一事业部、同一指标，不同年度的月度趋势叠加折线图 |
| 增长率排行 | 各事业部本年YTD同比增长率横向排行，一目了然增速差异 |
| 年度全景表格 | 各事业部 × 各指标的多年完成值矩阵，支持导出 |

## 5. 数据导入规范

### 5.1 通用规范

1. **文件格式**：支持 `.xlsx`、`.xls`、`.csv`（UTF-8编码）
2. **第一行**：列标题行（固定，不可修改列名）
3. **空行处理**：自动跳过空行，不报错
4. **数值格式**：金额列仅接受数字，不接受"万"等单位文字
5. **事业部名称**：必须与系统内名称完全一致（提供下拉选项模板）
6. **枚举字段**：必须使用模板内指定的枚举值（如"合同"而非"合同额"）

### 5.2 校验规则

| 校验项 | 规则 | 失败处理 |
|--------|------|----------|
| 必填字段 | 不可为空 | 该行失败，记录原因 |
| 年份格式 | 4位整数，范围2020-2040 | 该行失败 |
| 月份格式 | 1-12整数 | 该行失败 |
| 金额格式 | 非负数值 | 该行失败 |
| 事业部匹配 | 必须匹配系统内已有事业部名称 | 该行失败 |
| 指标类型 | 只能是"合同"/"收入"/"回款" | 该行失败 |
| 商机阶段 | 只能是指定枚举值 | 该行失败 |
| 重复数据 | 月度完成数据同组合覆盖写入，不报错 | 覆盖，提示数量 |

### 5.3 导入操作流程

```
1. 用户选择文件上传
2. 系统解析文件，展示预览（前5行）
3. 系统执行校验，展示校验摘要（有效行/无效行数量）
4. 用户确认后，系统执行导入
5. 系统返回导入结果（成功X条/失败X条）
6. 用户可下载失败详情报告
```

## 6. 非功能性需求

### 6.1 性能要求

| 场景 | 性能目标 |
|------|----------|
| 页面加载（首屏） | < 3秒（普通网络） |
| 导入1000行数据 | < 5秒（含校验+写入） |
| 看板图表渲染 | < 2秒 |
| 数据查询（列表） | < 1秒（当前数据规模） |

### 6.2 兼容性

- **浏览器**：Chrome 90+、Edge 90+（主流桌面版本）
- **分辨率**：1366×768 及以上（优先1920×1080设计）
- **操作系统**：Windows 10/11

### 6.3 可靠性

- 导入操作需为事务性操作：要么全部成功，要么失败后回滚，不允许部分写入导致数据不一致
- 年度目标、月度目标数据需有变更历史记录，防止数据意外丢失

### 6.4 数据安全

- V1.0 可采用简单的访问密码保护，防止数据外泄
- 不存储敏感个人信息
- 建议部署在内网环境

### 6.5 可维护性

- 事业部列表、指标类型等基础数据支持通过配置页面维护（而非硬编码），为未来扩展做准备
- 导入模板与系统字段保持同步，模板版本通过文件名标注

## 8. 验收标准

### 8.1 核心业务流程验收

| 验收场景 | 验收步骤 | 预期结果 |
|----------|----------|----------|
| 年度目标设置 | 在目标管理页为2026年各事业部录入三项指标目标 | 数据保存成功，页面正确展示 |
| 月度分解导入 | 上传月度分解模板文件（含12个月数据） | 导入成功，分解值正确写入 |
| 月度完成数据导入 | 上传月度完成数据（含多个事业部×多个月份） | 数据正确写入，覆盖提示符合预期 |
| 月度完成数据重复导入 | 再次上传相同月份数据（修改了部分数值） | 覆盖写入成功，旧值被替换 |
| 商机批量导入 | 上传商机导入模板（含有效行和无效行） | 有效行导入成功，无效行提示失败原因，可下载失败报告 |
| 总览看板数据正确性 | 录入数据后查看总览仪表盘 | KPI卡片显示正确，完成率计算无误 |
| 事业部钻取分析 | 从总览点击某事业部，进入详情页 | 月度趋势图、目标缺口分析数据正确 |
| 同比增长率分析 | 导入含历史年份数据后查看同比/趋势分析页 | 同比增长率计算正确，多年趋势图正常渲染 |

### 8.2 异常场景验收

| 验收场景 | 预期结果 |
|----------|----------|
| 导入文件格式错误（非xlsx/csv） | 系统提示文件格式不支持 |
| 导入文件中事业部名称拼写错误 | 该行标记失败，其余行正常导入 |
| 导入文件中金额列包含非数字字符 | 该行标记失败，提示错误原因 |
| 设置月度分解值总和超出年度目标 | 系统给出差异提示，但允许保存 |
| 无数据时访问看板 | 展示空状态提示，引导用户先导入数据 |

## 9. 附录：名词解释

| 术语 | 解释 |
|------|------|
| 合同目标 | 当年计划签署合同的金额总目标 |
| 收入目标 | 当年计划确认的营业收入总目标 |
| 回款目标 | 当年计划收回的现金款项总目标 |
| YTD | Year-to-Date，年初至今累计值 |
| 月度分解 | 将年度目标按月拆分后各月应完成的计划值 |
| 完成率 | 实际完成值 ÷ 目标值 × 100%，衡量目标达成程度 |
| 商机阶段 | 商机推进过程的节点状态：线索→立项→报价→签约跟进→已完成 |
| 同比增长率 | 本年YTD完成值与上年同期完成值的比较，衡量经营增速，上年数据通过导入历史文件获取 |
| 商机覆盖率 | 在手进行中商机总金额 / 目标缺口，衡量商机储备充裕度 |
